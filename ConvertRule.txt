웹페이지 변환 툴을 이용한 작성 요령
- 기본적으로 html, Bootstrap 코드를 그대로 인식한다.
- 특수한 기호들은 적절한 Bootstrap 코드로 변환한다.
- 페이지 리스트는 별도의 파일로 저장한다. (List.xlsx)
- 페이지 리스트를 기반으로 각 페이지는 카테고리와 페이지 순서 및 링크가 정해진다.
- 페이지는 메타 데이터를 포함하며, 3개의 언어가 한 페이지에 포함되어야 한다.
- 메타 데이터는 리스트 파일에 저장한다. (페이지 이름이자 ID, HTML의 제목 (3개의 언어), 상위 카테고리, 순서)

- 양식 데이터는 >타입:값으로 작성된다.
- 튜토리얼 페이지 외에는 직접 작성해보자

튜토리얼 페이지 양식 예시

>-------------------------------------<
: 구분자

// KOR //
: 한국어 페이지의 설명이다.

// ENG //
: 영어 페이지의 설명이다.

// JPN //
: 일본어 페이지의 설명이다.

(공통 부분)

>PATH
: "진입 경로". (AnyPortrait > Getting Started > Tutorial 1 > 2. Make Meshes)
: 내용과 링크는 메타 데이터를 기반으로 자동으로 생성되며, 값은 작성하지 않는다.

>TITLE
: 페이지의 메인 타이틀. 가장 큰 글씨
: 내용은 메타데이터를 기반으로 자동으로 생성된다.

>SUB_TITLE:텍스트
: 중간 제목. 텍스트를 입력해야한다.

>VERSION:1.2.4
: 이 페이지의 내용이 적용되는 버전. 숫자로 적어주자.

>FOOT_NAV
: 페이지 하단의 PREV, NEXT 링크.
: PREV, NEXT 여부 및 이름과 링크는 메타 데이터로부터 자동으로 생성한다.



(내용)

>TXT
...(내용)
: 일반 텍스트이다. 다음 타입명이 나올때까지 연속으로 파싱이 되므로 계속 작성하자.
: 이게 호출된 텍스트들은 <p>..</p>로 묶이며, 그 안에서의 줄바꿈은 <br>로 처리된다.
: 레이아웃과 유사하지만 END 처리는 따로 없다.

(특별한 텍스트 양식)
[양식1;양식2|텍스트] : 양식 타입간에 ;를 이용하여 구분한다.
["..."] : 타입이 아무것도 없으면 해당 텍스트에 Bold를 적용한다.
[B|"..."] : Bold 텍스트

[C_BLUE|"..."] : 파란색
[C_GREEN|"..."] : 녹색
[C_RED|"..."] : 붉은색
[C_YELLOW|"..."] : 노란색


[LINK<페이지 이름>|"..."] : 웹페이지 내로 이동하는 링크
[LINK_TAB<페이지 이름>|"..."] : 웹페이지 내로 이동하는 링크. 새 탭에서 열린다.
[LINK_EX<http:// 또는 https://..>|"..."] : 외부 페이지로 이동하는 링크
[LINK_EX_TAB<http:// 또는 https://..>|"..."] : 외부 페이지로 이동하는 링크. 새 탭에서 열린다.


(숫자) : 작업 순서에 관한 것이므로 자동으로 Bold가 적용된다, 숫자외의 글자가 있으면 적용 안됨


>SPACE
: 한줄 띄어쓴다. 코드로는 <br><br>


>IMAGE:(그림 파일 경로)
: 웹페이지에 포함된 그림 파일이다. 이미지 크기는 자동으로 설정된다.
: 경로는 "Images/"는 생략해도 되며, "서브 폴더/파일 이름"을 작성해야한다.
: 파일 확장자는 자동으로 검색한다.

>IMAGE_WIDTH:70:(그림 파일 경로)
: 이미지의 가로 크기를 웹페이지의 퍼센트로 지정한다.
: 일반 이미지보다 작은 경우 사용하는 코드로

>IMAGE_EXTERNAL:(외부 이미지 경로)
: 외부 이미지 파일을 가져온 경우의 링크이다.
: 클릭하면 "새탭에서" 링크가 열린다.


(이미지 파일 이름에 관한 규칙)
1) "Images/"와 "확장자"를 생략한다
2) Getting Started의 이미지인 경우 : GS_001 > GettingStarted/GS_001
3) Advanced Manual의 이미지인 경우 : ADV_001 > Advanced/AdManual_001
4) Advanced Manual의 썸네일인 경우 : ADVTMB_001 > AdvThumb/AdThumb_001


>YOUTUBE:(유튜브 링크)
: 유튜브 영상을 넣는다



(참고 박스)
>NOTE_START ~~ >NOTE_END
: 참고용 Alert 박스가 열리고 닫힌다. 이 사이의 코드들은 다르게 동작한다.
: 중간 제목이 조금 작게 보여진다.


(가로 레이아웃)
>HGRID_START ~>HGRID_MID ~ >HGRID_END
: 가로로 글, 그림을 배치한다.
: MID에서 좌, 우로 구분이 되며, 그 전에는 세로로 묶인다.


(스크립트 설명 박스)
>API_START:(함수명;함수명;...)
 ~~ >API_END
: 스크립트 설명용 박스가 열리고 닫힌다.
: >API_END 직후엔 무조건 <br>이 하나 붙는다.
: 함수명을 ;로 구분하면 오버로드된 것으로 간주하여 보여준다.
: 이 안에서는 다음의 양식만 사용할 수 있다.


>API_VER:1.2.4
: 버전 명을 미리 명시한다.(숫자)

>API_DESC
(스크립트 설명은 다음줄부터. TXT와 동일하게 처리)
: Description이라는 소제목과 함께 설명을 한다. 이 내부에서 한줄을 띄어쓰려면 그냥 엔터를 하나 더 치자.

>API_PARAM
(스크립트 설명은 다음줄부터. TXT와 동일하게 처리)
: Parameters라는 소제목과 함께 파라미터와 그 파라미터에 대한 설명을 한다.
: 이건 파라미터가 여러개인 경우 여러번 적으면 되며, 소제목을 한번만 붙는다.

>API_RETURN:(리턴 타입):(설명)
: Return이라는 소제목과 함께 리턴 타입과 그 설명을 한다.


(스크립트 작성 예시 박스)
>SCRIPT
: 이 다음부터 코드를 작성하면 되며, TXT와 유사하지만 코드에 맞게 변환을 한다.
: 스크립트 작성 박스이다.
: 이 안에서는 별도의 양식없이 코드를 적으면 된다.
: "주석", "reserved 구문"은 자동으로 색상이 지정된다.
: [...]를 이용하여 굵게 지정할 수 있다.
: 탭 문자는 &emsp; 두개로 표현한다.




(리스트 - 일반)
>LIST:(링크된 페이지 이름):(내용):(버튼 이름)
: 일반 리스트의 항목이다. 
: 제목과 링크는 페이지 이름으로 작성한다.

(리스트 - 썸네일 포함)
>LIST_IMAGE:(페이지 이름):(내용):(이미지 주소):(버튼 이름)
: 이미지 썸네일과 리스트 항목이다.
: 경로는 "Images/"는 생략하며, "서브 폴더/파일 이름"을 작성해야한다. (확장자 제외)
: 이미지는 "정사각형 비율"이어야 한다.
: 링크는 페이지 이름으로 검색한다.


(리스트 - 그리드)
>LIST_GRID_START ~~ >LIST_GRID_END
: 2단으로 구성되는 그리드 리스트를 시작한다.

>LIST_GRID_ITEM:(페이지 이름):(이미지 주소):(뱃지 UPDATED or NEW or 없음)
: 이미지 썸네일과 리스트 항목이다.
: 이미지 주소는 앞뒤 생략
: 만약 뱃지를 추가하고자 한다면 (페이지 이름)뒤에 ":UPDATED"나 ":NEW"를 붙이자




(검색 페이지 + 검색 테이블)
헤더 파일에 [SEARCH_SCRIPT] 부분에 자동으로 입력이 된다.

>SEARCH_GROUPS
: 검색 테이블로부터 데이터를 받아서 넣는다.

[검색 테이블]
- 페이지 이름 / 페이지 설명 / 검색 키워드들 (탭 문자로 구분)
